# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| latest  | :white_check_mark: |

We only support the latest release with security updates. Please upgrade to the latest version before reporting a vulnerability.

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

If you discover a security vulnerability, please report it privately to:

**[laishiwen7@gmail.com](mailto:laishiwen7@gmail.com)**

Please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Affected versions or components
- Any potential impact or exploit scenario

We will:

1. Acknowledge your report within **48 hours**
2. Provide an initial assessment within **5 business days**
3. Keep you informed of progress toward a fix
4. Credit you in the fix announcement (unless you prefer to remain anonymous)

## Disclosure Policy

We follow a coordinated disclosure process:

1. The reporter submits the vulnerability privately
2. We verify and develop a fix
3. We release the fix
4. We publish a security advisory after the fix is available

We ask that you not publicly disclose the vulnerability until we have had reasonable time to address it.

## Scope

Security-relevant areas include:

- Authentication and authorization
- Data exposure and privacy
- Input validation and sanitization
- API endpoints and parameter handling
- Dependency vulnerabilities
- Configuration and secret management

## Out of Scope

- Theoretical vulnerabilities without a practical proof of concept
- Issues that require physical access to the host machine
- Social engineering attacks
- Vulnerabilities in third-party services outside our control
